# 结构化业务数据 AI 问答模块需求说明

> 项目目标：新建一个独立 Python 项目，作为现有 RuoYi 系统之外的 AI 模块。该模块只处理现有 MySQL 结构化业务数据，不处理图片、OCR、PDF、Excel 文件解析。  
> 开发对象：Claude Code / Python 开发助手  
> 推荐技术栈：FastAPI + MySQL + OpenAI Compatible LLM API + Jinja2 SQL 模板 + 简单 Web 前端  
> 当前阶段：MVP 第一版，重点实现“自然语言 → 意图识别 → 查询模板 → SQL 执行 → AI 总结回答”。

---

## 1. 背景说明

现有业务系统基于 RuoYi / Spring Boot / MySQL，系统中已经存在厂家、产品、原料、出库、入库、月报、库存等结构化数据。

由于现有 RuoYi 系统较复杂，不希望直接在原系统中新增 AI 模块，因此需要新建一个独立 Python 服务，通过只读方式连接现有 MySQL 数据库，为前端或 RuoYi 提供 AI 查询接口。

本项目暂时不处理以下内容：

- 图片单据
- OCR 识别
- PDF 解析
- Excel 解析
- 文档上传知识库
- 向量化所有业务记录
- 自动写入、修改、删除业务表数据

本项目只处理：

- 现有 MySQL 结构化业务数据
- 业务表语义说明
- 字段说明
- 查询模板
- 业务规则
- 自然语言问答
- SQL 查询
- AI 总结分析

## 2. 第一版目标

第一版系统要做到：

用户输入：

```text
铭泰有哪些产品？
```

系统返回：

```text
铭泰塑胶颜料商行当前有以下产品：
1. 黄ABS，原料 ABS，单重 12，单价 13
2. 红PP，原料 PP，单重 10，单价 12

以上数据来自产品基础信息表，已过滤删除数据。
```

同时接口返回：

- 识别出的 intent
- 抽取出的 params
- 实际执行的 SQL
- SQL 查询结果
- AI 生成的自然语言回答
- 使用的查询模板
- 是否成功
- 错误信息
- 耗时

## 3. 核心思想

本项目不是传统文档 RAG，而是：

```text
结构化数据问答系统
= 业务语义层 + 查询模板 + SQL 安全执行 + LLM 总结
```

原则：

```text
能用 SQL 精确查询的，不使用向量库。
能用查询模板解决的，不让大模型自由生成 SQL。
大模型负责理解问题、抽参数、总结结果，不直接接管数据库。
```

## 6. 真实业务表结构读取要求

本项目不要在需求文档中硬编码或假设现有业务表结构。

开发时必须由程序或开发者直接从真实 MySQL 数据库读取表结构，然后再生成或修正 AI 元数据、查询模板和 SQL。

### 6.1 必须读取的真实表结构

第一版至少需要读取以下表的真实 DDL：

```sql
SHOW CREATE TABLE ad_factory_info;
SHOW CREATE TABLE ad_product_info;
SHOW CREATE TABLE ad_product_record;
SHOW CREATE TABLE ad_raw_record;
SHOW CREATE TABLE ad_month_inventory;
```

如果某些表在实际数据库中不存在，或者字段名与需求中的查询模板不同，必须以真实数据库为准修改模板。

### 6.3 自动导入表结构说明

建议实现一个开发辅助命令：

```bash
python scripts/inspect_schema.py
```

功能：

1. 连接 MySQL。
2. 执行 `SHOW CREATE TABLE`。
3. 输出所有目标表字段、类型、注释、索引。
4. 根据字段注释自动生成 `ai_table_schema` 和 `ai_field_schema` 的初始化 SQL 草稿。
5. 不自动覆盖已有业务规则，只生成待确认文件。

输出文件建议：

```text
sql/generated_schema_preview.sql
sql/schema_report.md
```

### 6.4 查询模板必须以真实字段为准

后文提供的查询模板 SQL 是业务逻辑样例，不代表一定能直接在当前数据库执行。

开发时必须检查：

- 产品表主键字段是否叫 `ad_product_info_id`
- 厂家外键是否叫 `ad_factory_info_id`
- 产品记录表中的数量字段是否叫 `num`
- 产品记录表中的重量字段是否叫 `weight`
- 产品记录表中的金额字段是否叫 `total_price`
- 时间字段是否叫 `record_time`
- 原料字段是否叫 `raw_materials_code` 或其他名称
- 月库存表是否已经存在，字段是否一致

如果不一致，必须修改：

```text
1. table_mapping.py
2. ai_field_schema 初始化数据
3. ai_query_template 中的 SQL 模板
4. Prompt 中的字段解释
```

## 7. AI 元数据表 SQL

### 7.1 表说明表：`ai_table_schema`

```sql
CREATE TABLE IF NOT EXISTS `ai_table_schema` (
  `id` BIGINT NOT NULL PRIMARY KEY,
  `table_name` VARCHAR(100) NOT NULL COMMENT '数据库表名',
  `business_name` VARCHAR(100) NOT NULL COMMENT '业务名称',
  `description` TEXT COMMENT '表的业务说明',
  `example_question` TEXT COMMENT '这个表可以回答的问题',
  `enabled` TINYINT DEFAULT 1 COMMENT '是否启用：1启用，0停用',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY `uk_table_name` (`table_name`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='AI表语义说明';
```

### 7.2 字段说明表：`ai_field_schema`

```sql
CREATE TABLE IF NOT EXISTS `ai_field_schema` (
  `id` BIGINT NOT NULL PRIMARY KEY,
  `table_name` VARCHAR(100) NOT NULL COMMENT '数据库表名',
  `field_name` VARCHAR(100) NOT NULL COMMENT '字段名',
  `business_name` VARCHAR(100) NOT NULL COMMENT '字段中文含义',
  `description` TEXT COMMENT '字段说明',
  `value_mapping` JSON COMMENT '枚举值说明',
  `example_value` VARCHAR(255) COMMENT '示例值',
  `enabled` TINYINT DEFAULT 1 COMMENT '是否启用',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY `uk_table_field` (`table_name`, `field_name`),
  KEY `idx_table_name` (`table_name`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='AI字段语义说明';
```

### 7.3 业务规则表：`ai_business_rule`

```sql
CREATE TABLE IF NOT EXISTS `ai_business_rule` (
  `id` BIGINT NOT NULL PRIMARY KEY,
  `rule_name` VARCHAR(100) NOT NULL COMMENT '规则名称',
  `rule_type` VARCHAR(50) COMMENT '规则类型：unit/inventory/month_report/query/filter',
  `rule_content` TEXT NOT NULL COMMENT '规则内容',
  `related_tables` VARCHAR(500) COMMENT '相关表，逗号分隔',
  `priority` INT DEFAULT 0 COMMENT '优先级，越大越重要',
  `enabled` TINYINT DEFAULT 1 COMMENT '是否启用',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  KEY `idx_rule_type` (`rule_type`),
  KEY `idx_enabled` (`enabled`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='AI业务规则';
```

### 7.4 查询模板表：`ai_query_template`

```sql
CREATE TABLE IF NOT EXISTS `ai_query_template` (
  `id` BIGINT NOT NULL PRIMARY KEY,
  `template_name` VARCHAR(100) NOT NULL COMMENT '模板名称',
  `intent_code` VARCHAR(100) NOT NULL COMMENT '意图编码',
  `description` TEXT COMMENT '模板说明',
  `sql_template` TEXT NOT NULL COMMENT 'SQL模板，使用 Jinja2 变量语法',
  `param_schema` JSON COMMENT '参数说明',
  `example_question` TEXT COMMENT '示例问题',
  `result_description` TEXT COMMENT '结果字段说明',
  `enabled` TINYINT DEFAULT 1 COMMENT '是否启用',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY `uk_intent_code` (`intent_code`),
  KEY `idx_enabled` (`enabled`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='AI查询模板';
```

### 7.5 聊天记录表：`ai_chat_log`

```sql
CREATE TABLE IF NOT EXISTS `ai_chat_log` (
  `id` BIGINT NOT NULL PRIMARY KEY AUTO_INCREMENT,
  `session_id` VARCHAR(100) DEFAULT NULL COMMENT '会话ID',
  `user_id` VARCHAR(100) DEFAULT NULL COMMENT '用户ID，可选',

  `sql_rows_json` JSON COMMENT 'SQL结果，注意控制长度',
  `answer` LONGTEXT COMMENT 'AI回答',
  `success` TINYINT DEFAULT 1 COMMENT '是否成功',
  `error_msg` TEXT COMMENT '错误信息',
  `model_name` VARCHAR(100) DEFAULT NULL COMMENT '模型名称',
  `duration_ms` INT DEFAULT NULL COMMENT '耗时毫秒',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  KEY `idx_session_id` (`session_id`),
  KEY `idx_intent_code` (`intent_code`),
  KEY `idx_create_time` (`create_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='AI问答记录';
```

## 9. 查询模板设计

第一版不要让大模型自由写 SQL。只允许选择固定 intent，然后渲染固定 SQL 模板。

### 9.1 支持的 intent

```text
factory_product_list        查询厂家产品列表
product_info_query          查询产品基础信息
product_out_summary         查询产品出货汇总
raw_material_usage          查询原料使用汇总
monthly_inventory_query     查询月度库存
business_rule_explain       解释业务规则
unknown                     无法识别
```

### 9.2 查询厂家产品列表

```sql
INSERT INTO ai_query_template
(id, template_name, intent_code, description, sql_template, param_schema, example_question, result_description, enabled)
VALUES
(1,
'查询厂家产品列表',
'factory_product_list',
'根据厂家名称查询该厂家下的产品列表',
'SELECT
   afi.factory_info_id,
   afi.factory_name,
   CASE WHEN afi.is_kg = 1 THEN ''KG'' ELSE ''斤'' END AS weight_unit,
   api.ad_product_info_id,
   api.ad_product_name,
   api.color,
   api.raw_materials_code,
   api.weight,
   api.unit_price
 FROM ad_product_info api
 JOIN ad_factory_info afi ON api.ad_factory_info_id = afi.factory_info_id
 WHERE api.del_flag = ''0''
```

### 11.2 SQL 模板渲染

要求：

- 使用 Jinja2。
- SQL 模板中变量使用 `:paramName` 绑定参数格式。
- 不要直接字符串拼接用户输入。
- 渲染后通过 SQLAlchemy 执行参数绑定。

```python
from jinja2 import Template


def render_sql_template(sql_template: str, params: dict):
    template = Template(sql_template)
    rendered_sql = template.render(**params)

    bind_params = {}
    for key, value in params.items():
        if value is not None and value != "":
            bind_params[key] = value

    return rendered_sql, bind_params
```

### 11.3 SQL 安全检查

文件建议：`app/core/sql_safety.py`

必须实现：

1. 只允许 SELECT 或 WITH 开头。
2. 禁止多语句。
3. 禁止危险关键字。
4. 禁止访问非白名单表。
5. 自动限制返回行数。
6. 不允许 `SELECT *`。
7. 可配置 SQL 超时时间。

示例：

```python
import re
import sqlparse

FORBIDDEN_KEYWORDS = [
    "insert", "update", "delete", "drop", "alter", "truncate",
    "create", "replace", "grant", "revoke", "call", "exec",
    "merge", "load", "outfile", "dumpfile"
]

ALLOWED_TABLES = {
    "ad_factory_info",
    "ad_product_info",
    "ad_product_record",
    "ad_raw_record",
    "ad_month_inventory",
    "ai_table_schema",
    "ai_field_schema",
    "ai_business_rule",
    "ai_query_template"
}


def check_sql_safety(sql: str):
    if not sql or not sql.strip():
        raise ValueError("SQL 不能为空")

    stripped = sql.strip()
    lower = stripped.lower()

    if ";" in stripped.rstrip(";"):
        raise ValueError("不允许多语句 SQL")

    if not (lower.startswith("select") or lower.startswith("with")):
        raise ValueError("只允许 SELECT 查询")

    for kw in FORBIDDEN_KEYWORDS:
        if re.search(rf"\b{kw}\b", lower):
            raise ValueError(f"SQL 包含危险关键字：{kw}")

    if re.search(r"select\s+\*", lower):
        raise ValueError("不允许使用 SELECT *，请明确查询字段")

    parsed = sqlparse.parse(sql)
    if len(parsed) != 1:
        raise ValueError("只允许单条 SQL")

    for match in re.finditer(r"\bfrom\s+([a-zA-Z0-9_]+)|\bjoin\s+([a-zA-Z0-9_]+)", lower):
        table = match.group(1) or match.group(2)
        if table and table not in ALLOWED_TABLES:
            raise ValueError(f"不允许访问表：{table}")

    return True
```

### 11.4 查询执行

要求：

- 使用只读数据库账号。
- 默认最大返回 200 行。
- 查询超时建议 10 秒。
- 返回 dict list。
- Decimal、datetime 要转 JSON 友好格式。

示例：

```python
from decimal import Decimal
from datetime import datetime, date


def normalize_value(value):
    if isinstance(value, Decimal):
        return float(value)
    if isinstance(value, datetime):
        return value.strftime("%Y-%m-%d %H:%M:%S")
    if isinstance(value, date):
        return value.strftime("%Y-%m-%d")
    return value


def normalize_rows(rows):
    result = []
    for row in rows:
        result.append({k: normalize_value(v) for k, v in dict(row).items()})
    return result
```

## 17. 结果格式化要求

SQL 查询结果中可能出现：

- Decimal
- datetime
- date
- bytes
- null

需要转换成 JSON 可序列化格式。

规则：

```text
Decimal → float 或 string
datetime → yyyy-MM-dd HH:mm:ss
date → yyyy-MM-dd
bytes → utf-8 string 或 base64
null → null
```

金额建议保留 2 位小数。  
重量建议保留 4 位小数。  
数量建议根据实际情况保留 2 或 4 位小数。

## 18. 日期处理要求

当前时间应从服务器获取。

示例：如果当前日期是 `2026-05-11`：

```text
这个月：
startDate = 2026-05-01 00:00:00
endDate   = 2026-06-01 00:00:00
monthStr  = 2026-05

上个月：
startDate = 2026-04-01 00:00:00
endDate   = 2026-05-01 00:00:00
monthStr  = 2026-04

今天：
startDate = 2026-05-11 00:00:00
endDate   = 2026-05-12 00:00:00
```

LLM 负责初步识别，后端也应该二次修正常见相对日期。

建议实现：

```python
resolve_relative_date_params(params: dict, question: str) -> dict
```

如果用户问题中包含：

- 这个月
- 本月
- 上个月
- 上月
- 今天
- 昨天
- 今年

后端应优先保证日期参数正确。
